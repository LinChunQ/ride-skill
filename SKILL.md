# RIDE Flow 开发工作流

你正在协助用户完成一个结构化的开发流程，该流程遵循四个顺序阶段：**R**需求对齐 → **I**实施规划 → **D**开发 → **E**导出API文档。

## 核心原则

- 严格按顺序执行各阶段（R → I → D → E）
- 未经用户明确同意，不得进入下一阶段
- 维护与代码变更同步的活文档
- 最终产出前端可直接使用的API规范文档
- **重要**：所有文档、注释、代码说明均使用中文

## 触发方式

当用户说出以下任一表述时，立即启动此工作流：
- "使用 RIDE 工作流"
- "启动 RIDE"
- "按照 RIDE 流程"
- "RIDE 开发"
- 或任何明确提到 "RIDE" 的请求

## 初始化

当用户启动此工作流时，立即执行以下初始化步骤：

1. 生成项目目录名称：使用当前时间戳和需求关键词，格式为 `YYYYMMDD_HHMM_需求简述`（例如：`20260525_1530_用户认证`）
2. 在 `docs/` 目录下创建该项目目录
3. 在项目目录下创建以下文件：
   - `需求说明.md` - 用于记录需求对齐阶段的所有信息
   - `实施方案.md` - 用于记录实施规划和开发过程
   - `API对接文档.md` - 用于记录最终的前端API规范
4. 在 `需求说明.md` 中写入初始内容：
   ```markdown
   # 需求说明
   
   ## 原始需求
   [用户提供的需求描述]
   
   ## 需求澄清
   [记录需求对齐阶段的问答]
   
   ## 最终确认需求
   [整理后的完整需求描述]
   ```

完成初始化后，开始执行阶段 [R]。

---

## 阶段 [R]: 需求对齐

**目标**：在编写任何代码之前，澄清所有实施细节。

**执行动作**：
1. 将用户提供的原始需求写入 `需求说明.md` 的"原始需求"部分
2. 仔细阅读用户提供的初始业务需求
3. 分析当前项目的技术栈（后端框架、数据库、架构模式）
4. 针对实施细节提出 3-5 个有针对性的问题，重点关注：
   - 并发场景下的数据一致性
   - 数据库表结构的可扩展性
   - 错误处理和重试机制
   - 缓存策略
   - 边界条件和异常情况
5. 将问答过程记录到 `需求说明.md` 的"需求澄清"部分
6. 需求完全明确后，整理完整的需求描述写入"最终确认需求"部分

**约束条件**：
- 本阶段**禁止编写任何业务逻辑代码**
- 持续提问直到获得足够的细节来创建完整的实施方案

**阶段流转**：仅当需求完全理解后，才请求用户批准进入阶段 [I]。

## 阶段 [I]: 实施规划

**目标**：创建全面的实施方案文档。

**执行动作**：
1. 在项目目录的 `实施方案.md` 文件中编写详细的实施方案
2. 文档必须包含以下结构化章节：
   - **架构与流程**：核心业务逻辑流程图（使用 Mermaid 语法）
   - **数据模型**：详细的数据库表结构变更说明（字段类型、索引、约束）
   - **代码结构**：核心逻辑的伪代码或骨架设计
   - **API 契约**：后端 API 接口定义（路由、方法、请求/响应格式）

**输出要求**：
- 文档结构清晰，层次分明
- 细节充分，实施时无歧义
- 包含技术决策及其理由
- **所有内容使用中文编写**

**阶段流转**：等待用户审核和批准后，才能进入阶段 [D]。

## 阶段 [D]: 开发

**目标**：实施功能的同时保持文档与代码同步。

**执行动作**：
1. 协助完成具体的编码任务（Controller、Service、Repository 等）
2. 为新功能编写测试
3. 遵循阶段 [I] 中创建的实施方案
4. **所有代码注释使用中文**

**活文档规则**：
- 当技术障碍导致需要修改原始逻辑、数据库表结构或 API 契约时，**立即自动更新 `实施方案.md`**
- 始终保持文档与实际代码 100% 同步
- 在 `实施方案.md` 末尾添加”变更记录”章节，记录任何偏离原始方案的变更及其原因

**阶段流转**：当用户请求”导出前端文档”或”进入 E 阶段”时，进入阶段 [E]。

## 阶段 [E]: 导出 API 文档

**目标**：生成前端可直接使用的 API 文档。

**触发条件**：用户明确请求”导出前端文档”或”进入 E 阶段”。

**执行动作**：
1. 扫描项目中相关的 Controller/Router 层代码
2. 识别关联的 Entity/DTO（数据传输对象）类
3. 将 API 规范写入项目目录的 `API对接文档.md` 文件

**文档要求**：
- 精确的 JSON 请求/响应示例
- 明确的字段类型（String、Integer、Boolean 等）
- 必填字段验证规则
- 枚举值定义
- 零歧义，便于 AI 解析
- **所有文档内容使用中文**

**目标受众**：前端开发者及其 AI 助手。文档必须可被解析以生成 TypeScript 接口或类似的类型定义。

## 使用方法

### 在支持 skill 的工具中（如 Claude Code）
使用命令启动（如 `/ride`）。

### 在不支持 skill 的工具中（如 Codex、Cursor 等）
直接对 AI 说：
- "使用 RIDE 工作流来实现 [功能描述]"
- "启动 RIDE 流程，需求是：[需求描述]"
- "按照 RIDE 开发流程来做 [功能]"

工作流将从阶段 [R] 开始，并按顺序引导你完成每个阶段。
